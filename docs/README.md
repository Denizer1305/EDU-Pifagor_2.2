# Docs — EDU-Pifagor

Папка `docs/` предназначена для архитектурной, технической и дипломной документации проекта **«Пифагор»**.

## Что хранить в документации

Рекомендуемые разделы:
- ER-диаграммы;
- описания доменной модели;
- архитектурные решения;
- диаграммы use case;
- схемы развертывания;
- материалы дипломного проектирования;
- текстовые пояснения по ключевым модулям.

## Предлагаемая структура

```text
docs/
├── README.md
├── README.en.md
├── architecture/
│   ├── er-diagram.md
│   ├── domain-model.md
│   ├── api-architecture.md
│   └── decisions/
├── diagrams/
│   ├── er/
│   ├── use-cases/
│   └── deployment/
└── diploma/
```

## Назначение документации

Документация нужна для того, чтобы:
- быстро погружаться в проект;
- передавать проект другому разработчику;
- фиксировать принятые решения;
- использовать материалы в дипломе;
- поддерживать единое понимание архитектуры.
