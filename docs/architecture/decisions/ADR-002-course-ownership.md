# ADR-002 — Один курс принадлежит одному преподавателю

## Контекст

На раннем этапе рассматривалась модель, в которой один курс могли вести несколько преподавателей.

## Решение

Зафиксировать правило:
- один курс принадлежит одному преподавателю;
- один преподаватель может вести много курсов;
- чужие курсы не редактируются другими преподавателями.

Академическая связь строится через `TeachingAssignment`.

## Последствия

### Плюсы
- проще разграничение прав;
- проще аудит изменений;
- понятнее бизнес-логика;
- проще интерфейс.

### Минусы
- меньше гибкости для совместного преподавания;
- совместную работу придется реализовывать дополнительными механизмами при необходимости.
