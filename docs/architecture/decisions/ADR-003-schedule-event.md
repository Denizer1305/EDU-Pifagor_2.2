# ADR-003 — Универсальная сущность ScheduleEvent

## Контекст

Изначально расписание можно было моделировать как набор простых записей занятий.

Однако платформа требует:
- регулярные занятия;
- контрольные;
- заметки преподавателя;
- напоминания;
- события календаря.

## Решение

Использовать единую сущность `ScheduleEvent` как базовую модель для календаря и расписания.

## Последствия

### Плюсы
- единая модель событий;
- проще сопровождение;
- легче развивать календарный модуль;
- меньше дублирования.

### Минусы
- сущность становится шире;
- требуется аккуратная типизация событий и ограничений.
