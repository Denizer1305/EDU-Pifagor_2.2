# ADR-001 — Монорепозиторий

## Контекст

Проект включает:
- backend;
- frontend;
- deploy;
- docs.

Нужна структура, удобная для одного разработчика и дипломного проекта.

## Решение

Использовать **монорепозиторий**:
- `backend/`
- `frontend/`
- `deploy/`
- `docs/`

## Последствия

### Плюсы
- единая точка входа;
- проще синхронизация frontend и backend;
- проще запуск и документация;
- удобно для диплома.

### Минусы
- репозиторий растет быстрее;
- при большой команде может потребоваться более строгая организация процессов.
