# Domain Model

## Общая идея

«Пифагор» — это LMS-платформа, в которой основными участниками являются:
- администратор;
- преподаватель;
- студент;
- родитель.

Система строится вокруг учебного процесса, а не вокруг отдельных экранов интерфейса.

## Центральные домены

### 1. Пользователи
Отвечают за учетные записи, роли, профили и связи между родителями и студентами.

### 2. Организации
Позволяют учитывать разные образовательные учреждения, формы обучения, предметы и группы.

### 3. Академические назначения
`TeachingAssignment` связывает преподавателя, предмет, группу и учебный год.

### 4. Курсы
Курс — это цифровая оболочка академического назначения, в которой находятся модули, уроки, материалы, задания и тесты.

### 5. Контроль знаний
Система поддерживает задания, тестирование, проверки, журнал и посещаемость.

### 6. Расписание
`ScheduleEvent` — универсальная сущность для занятий, контрольных, напоминаний и иных календарных событий.

### 7. Аналитика
Аналитика строится через запросы к БД и вычисление показателей в сервисном слое.

## Модуль должников

Ключевая управленческая функция аналитики — выявление студентов с академическими задолженностями:
- просроченные задания;
- несданные работы;
- низкий процент выполнения;
- слабая учебная динамика.

## Принципы моделирования

- не дублировать вычисляемые данные;
- не создавать таблицу под каждый экран;
- выделять отдельные сущности только для реально самостоятельных объектов предметной области;
- фиксировать архитектурные решения в ADR-документах.
