# Docs — EDU-Pifagor

The `docs/` directory is предназначена for architectural, technical, and graduation-project documentation for **Pifagor Educational Platform**.

## What to Store in Documentation

Recommended sections:
- ER diagrams;
- domain model descriptions;
- architecture decisions;
- use case diagrams;
- deployment schemes;
- graduation project materials;
- textual explanations for key modules.

## Suggested Structure

```text
docs/
├── README.md
├── README.en.md
├── architecture/
│   ├── er-diagram.md
│   ├── domain-model.md
│   ├── api-architecture.md
│   └── decisions/
├── diagrams/
│   ├── er/
│   ├── use-cases/
│   └── deployment/
└── diploma/
```

## Purpose of Documentation

Documentation is needed to:
- understand the project quickly;
- hand the project over to another developer;
- record key architectural decisions;
- reuse materials in the graduation project;
- keep a shared understanding of the system architecture.
